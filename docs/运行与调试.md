# 运行与调试

## 先记住一件事

这个仓库里的源码，并不会自动成为 OpenClaw 正在运行的插件。

真正被 OpenClaw 加载的通常是：

```text
~/.openclaw/extensions/tokenpilot/dist/index.js
```

所以“本地 build 成功”不等于“OpenClaw 已经在跑新代码”。

## 推荐的本地验证顺序

### 1. 构建

```bash
pnpm build
pnpm -r typecheck
```

### 2. 同步插件产物

最常见的是把下面几个文件同步到 OpenClaw 插件目录：

- `packages/openclaw-plugin/dist/index.js`
- `packages/openclaw-plugin/dist/index.js.map`
- `packages/openclaw-plugin/openclaw.plugin.json`

### 3. 校验配置

```bash
OPENCLAW_CONFIG_PATH=$HOME/.openclaw/openclaw.json openclaw config validate
```

### 4. 重启 gateway

```bash
openclaw gateway restart
tail -n 100 $HOME/.openclaw/logs/gateway.log
```

## 重点日志和状态目录

默认状态目录通常在：

```text
$HOME/.openclaw/tokenpilot-plugin-state/tokenpilot/
```

重点文件：

- `provider-traffic.jsonl`
- `task-state/trace.jsonl`
- `task-state/<session>/registry.json`
- `task-state/<session>/raw-turns/*.json`

## 遇到问题先看哪里

### 请求行为不对

先看：

- `proxy-runtime.ts`
- `task-state/trace.jsonl`
- gateway log

### reduction 没生效

先看：

- `before-call-reduction.ts`
- `runtime-core/src/passes/*`
- 配置里的 `reduction.passes`

### estimator 没跑或跑偏

先看：

- `policy.ts`
- `task-state-estimator.ts`
- `task-state/<session>/registry.json`
- `task-state/estimator-output.jsonl`

### recovery 不工作

先看：

- `runtime-core/src/archive-recovery/index.ts`
- `page-in/recovery-tool.ts`
- `TOKENPILOT_MEMORY_FAULT_RECOVERY_ENABLED`

## smoke 脚本

保留的脚本：

- `docs/scripts/smoke_isolated_gateway.sh`

用途：

- 起一个隔离的 OpenClaw gateway
- 用本地配置做 smoke test

建议把它当作“调通链路”的工具，而不是 benchmark 主入口。
